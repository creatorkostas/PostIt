PostIt Documentation

Your local-first, Git-friendly API client & test automation tool.

1. Navigation & Ecosystem

Global Search & Navigation Pro

Press Ctrl+P anywhere in the app to open the Quick Jump menu. Type a path or method to instantly navigate across your entire workspace.

Environment Switcher & The Vault Pro

Manage different stages (e.g., Local, Staging, Prod) using the ENV dropdown in the toolbar. Click Environments to configure variables per stage.

  • The Vault: Mark variables as secret (🔒). They are encrypted via AES-GCM and saved to disk safely. You must unlock the vault with your master password to use them.

Import & Export

  • Import Paste a cURL command to instantly generate a full request profile.
  • Publish Click this to automatically generate and host a beautiful, static HTML documentation page for all your requests.

2. Request Editor & Automation

Magic Variables Pro

Instantly generate dynamic data in URLs, Headers, or Body by using double curly braces. Supported magic variables include:

  • {{$guid}} - Generates a UUID v4
  • {{$timestamp}} - Current Unix timestamp
  • {{$isoTimestamp}} - Current ISO-8601 time
  • {{$randomInt}} - Random integer (0-1000)
  • {{$randomPassword}} - Generates a secure 12-character string

Standard Configuration

  • Body: Supports raw (JSON/Text) and x-www-form-urlencoded. Click Beautify JSON to format raw bodies.
  • Scripts: Write JS to run before or after requests. Supports gjson data extraction.

1. Request & Response History New

Persistent Snapshots & Ordering

Every request sent via the Web UI, TUI, or CLI is now automatically recorded in the HISTORY tab with its full context.

  • Absolute Ordering: Collections now maintain their exact import sequence. Whether loading from disk or importing a new Postman file, your requests stay exactly where you put them.
  • Full Response Saving: Unlike traditional clients, PostIt saves the exact response body and headers for every historical call.
  • Time Travel: Click any item in the History sidebar to instantly restore that specific response snapshot, even if the server's data has since changed.
  • Metadata: Each record includes the precise timestamp, method, resolved URL, status code, and millisecond duration.

3. Response & Validation

Advanced JSON Transformation Engine Pro

PostIt features a powerful query engine in the response FILTER bar. You can drill down, search, filter, and transform JSON data on the fly.

Multi-Filtering & Exclusions New

You can now combine multiple filters and exclusions in a single line, separated by spaces.

  • Multiple Exclusions: Hide several deep keys at once.
    -**.literals -**.assets -metadata
  • Combined Query: Exclude data first, then run a path query.
    -**.debugInfo data.users[active=true]

Filtering & Searching

  • Drill Down: Standard dot notation.
    data.user.profile
  • Exclude: Hide specific keys using the minus prefix. Supports deep search.
    -data.logs or -**.password
  • Deep Search: Find a key anywhere in the JSON hierarchy.
    **.id (Returns all IDs found at any level)
  • Predicates: Filter arrays by property values. Supports =, !=, >, <, >=, <=.
    data.users[id=5] or products[price>100]

Projections & Formatting

  • Field Projection: Pick specific fields to clean up the view.
    data.users{id, username, email}
  • Flatten View: Convert nested objects into a single-level key-value map.
    data | flatten

Transformation Pipes

Chain transformations using the pipe (|) character:

  • | count - Returns the number of items in an array or keys in an object.
  • | first / | last - Returns the first or last item of an array.
  • | sort:key - Sorts an array of objects by the specified key.
  • | keys - Returns a list of all top-level keys in an object.

Example Advanced Queries

  • Exclude & Count: -**.metadata data.items | count
  • Deep Find & First: **.emails | first
  • Complex Chain: data.users[role=admin]{id, name} | sort:name

Standard Analysis

  • JWT Tab: Automatically detects, extracts, and decodes any JSON Web Tokens found in the response body or headers.
  • Diff Tab: Compare your current response side-by-side with a pinned Baseline.

4. Advanced Power Tools Pro

The Hammer (Load Testing)

Switch to the Hammer tab to stress-test the current request. Set your Concurrency (Workers) and Duration.

  • Live Dashboard: View total requests, RPS (Requests Per Second), and average latency in real-time.
  • Percentiles: Automatically calculates strict P95 and P99 latencies.
  • Chart.js Integration: Watch a live graphical plot of your API's performance stability.

Database Sidekick (SQL)

Run SQL queries immediately after an API call to verify the data layer. Supports SQLite, PostgreSQL, and MySQL.

  • Extract to Var: Pull a specific cell from your SQL result (e.g., column id) and save it directly to an Environment Variable (e.g., user_id) for the next API call.

Visual Workflows

Orchestrate multi-step API sequences via the WORKFLOWS sidebar tab. Features an interactive Drag-and-Drop SVG Canvas.

  • Request Nodes: Fire API calls and extract data using GJSON paths into variables.
  • Condition Nodes (If/Else): Branch execution (Success/Failure lines) based on GJSON evaluations of the previous response.
  • Loop Nodes: Point to a JSON array. The engine will loop the attached sub-chain for every item, injecting the current payload as {{$item}}.
  • Wait Nodes: Introduce artificial delays between calls.

The Observer (Proxy Mode) Pro

PostIt can act as an HTTP Proxy to intercept and record traffic from your browser or mobile applications.

  • Start: Click 📡 Start Proxy in the toolbar and choose a port (default 8081).
  • Intercept: Configure your application or OS to use PostIt as its HTTP proxy. All outgoing traffic will be cloned and saved as new requests.
  • Auto-Refresh: The sidebar automatically updates every 5 seconds while the proxy is running, showing newly captured requests under Intercepted > [Date].

Intelligent Mocks

Start PostIt with ./postit.exe -ui web -mock to spin up a local server on /mock/.

  • Dynamic Routing: Define paths like /users/:id. PostIt extracts :id and exposes it as a variable for the mock response.
  • Conditional Mocks: Save multiple mocks for one route. Define a GJSON condition (e.g., id == "5"). The mock server evaluates the incoming request body and serves the correct dynamic response.
  • Simulated Latency New: Add a Delay (ms) to any mock to simulate slow network conditions or long-running operations.
  • Mock Management New: View and delete saved mocks directly from the Mocks tab in the request editor.

Real-time Communication (WebSockets) New

Connect to and interact with WebSocket servers directly from the WS sidebar tab.

  • Interactive Log: View incoming and outgoing messages with precise timestamps and color-coding.
  • Persistent Connection: Keep the connection open while you navigate other parts of the app.

CLI Runner (CI/CD)

Execute your collections entirely headlessly from the terminal for automation pipelines:

./postit.exe run --collection ./output/history.json --env ./output/environments.json --delay 500

The CLI runner respects the Absolute Order of your collection, ensuring tests run in the exact sequence defined in your workspace.
