PostIt Documentation

Your local-first, Git-friendly API client guide.

1. Navigation Sidebar

Search Bar

Type anywhere in the search bar at the top to instantly filter your collections and history. Folders will automatically expand to show matching requests.

Tabs: Collections vs History

  • Collections: View your saved requests organized by folders. Right-click a request to move it up/down or duplicate it.
  • History: A chronological log of every request sent. Click an item to reload it, or use the ✕ to delete specific entries.

Action Buttons

  • Vars Manage global environment variables used in {{brackets}}.
  • New Create a new request path (e.g., Folder > My API).
  • Edit Rename or move the selected request.
  • Copy Quickly duplicate the selected request configuration.
  • Clear (History Tab only) Wipe your entire request history.

2. Request Editor

The Toolbar

  • Save Changes Persists Method, URL, Body, and Scripts to your local .json file.
  • Code Snippet New Generates ready-to-use code for Go, Python, or JavaScript.
  • Method Selector: Choose between GET, POST, PUT, DELETE, and PATCH.
  • URL Input: Supports dynamic variables (e.g., {{host}}/api/login).

Configuration Tabs

  • Body: Supports raw (JSON/Text) and x-www-form-urlencoded modes.
  • Headers: Add custom key-value pairs for your request.
  • Mocks: New View saved mock responses for this specific path and see your local Mock URL.
  • Scripts: Write JavaScript to run before (Pre-request) or after (Post-request) the execution.

3. Response & Analysis

Analysis Tabs

  • Body: View the raw response. Includes a Filter Bar at the top—type data.id to drill into JSON instantly.
  • Headers: View all headers returned by the server.
  • Preview: New Renders HTML pages in an iframe or displays images directly.
  • Diff: New Compare your current response side-by-side with a saved Baseline.

Post-Execution Actions

  • Set as Baseline Pins the current response as the reference point for the Diff view.
  • Save as Mock Saves the current response body, status, and headers as a mock template.

4. Advanced Features

Mock Server

To enable the local mock server, start PostIt with the -mock flag:

./postit.exe -ui web -mock

Once enabled, you can call http://localhost:8080/mock/your-api-path to receive your saved mock responses.

Git-Friendly Storage

PostIt saves every request as a separate, pretty-printed JSON file in the output/ directory. This makes it easy to track changes in Git and collaborate with your team without proprietary cloud syncing.
