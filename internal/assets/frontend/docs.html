1. Overview & Getting Started

What is PostIt?

PostIt is a local-first, Git-friendly API client and test automation tool. It stores all your requests, environments, and history as plain JSON files on disk, making them naturally version-controllable with Git. No cloud dependency, no vendor lock-in.

Starting the Application

Launch the Web UI with:

./postit.exe -ui web -port 8080

Additional flags:

  • -ui web|tui|cli — Choose interface (default: web)
  • -port 8080 — Port for the Web UI
  • -mock — Enable the mock server at /mock/
  • -import path1,path2 — Import Postman collections on startup

Once started, open http://localhost:8080 in your browser.

Data Storage

All data is stored under the output/ directory:

  • output/requests/ — Individual request JSON files
  • output/collection.json — Reconstructed Postman-compatible collection
  • output/variables.json — Global variables
  • output/history.json — Request history
  • output/environments.json — Environment definitions
  • output/workflows.json — Visual workflows
  • output/global_headers.json — Global request headers
  • output/active_env.json — Active environment ID

Because everything is plain JSON, you can commit your output/ directory to Git and share your entire workspace with your team.

2. Navigation & Ecosystem

Global Search (Ctrl+P)

Press Ctrl+P anywhere in the app to open the Quick Jump menu. Type a path or method (e.g., GET, /users) to instantly navigate to any request across your entire workspace.

Command Palette (Ctrl+Shift+P)

Press Ctrl+Shift+P to open the Command Palette. Rapidly access all app functions:

  • Beautify JSON — Format the request body
  • Toggle Environment — Switch environments
  • Toggle Proxy — Start/stop the HTTP proxy
  • Import cURL — Open the import dialog
  • Export Collection — Download the collection

Sidebar Tabs

The left sidebar organizes your workspace into tabs:

  • COLLECTIONS — Your request tree with folder structure
  • HISTORY — Past request snapshots with response data
  • WORKFLOWS — Visual multi-step workflow canvas
  • WS — WebSocket client
  • MOCK — Mock server statistics dashboard

Sidebar Toolbar

Above the sidebar tabs, quick-action buttons provide access to:

  • Book Open icon — Open this documentation page
  • External Link icon — Generate & download API docs
  • Database icon — View/edit global variables
  • Download icon — Import cURL command
  • Plus icon — Create a new request

Context Menu (Right-Click)

Right-click any request in the sidebar to access:

  • Move Up / Move Down — Reorder requests
  • New Request — Create a sibling request
  • Rename / Move — Change the path of a request
  • Duplicate — Clone a request
  • Export Postman Collection — Download selected folder
  • Delete — Remove the request

3. Request Editor

URL Bar

The top URL bar lets you select an HTTP method (GET, POST, PUT, DELETE, PATCH) and enter the request URL. Click Send to execute the request.

Magic Variables Dynamic

Instantly generate dynamic data in URLs, Headers, or Body using double curly braces:

  • {{$guid}} — Generates a UUID v4
  • {{$timestamp}} — Current Unix timestamp
  • {{$isoTimestamp}} — Current ISO-8601 time
  • {{$randomInt}} — Random integer (0-1000)
  • {{$randomPassword}} — Generates a secure 12-character string
  • {{variable_name}} — References a global or environment variable

Body Modes

  • raw — JSON or plain text body with syntax highlighting. Click Beautify JSON to format.
  • x-www-form-urlencoded — Key-value pairs for form submissions
  • GraphQL — Dedicated editor with separate query and variables panels

Headers Tab

Add, edit, or remove request headers. Supports variable resolution (e.g., {{auth_token}}). Global headers are automatically merged with per-request headers.

Pre-request Scripts

Write JavaScript to execute before the request is sent. Supports:

  • pm.collectionVariables.set("key", value)
  • pm.environment.set("key", value)
  • pm.globals.set("key", value)
  • Variable access via {{variable}} in the script

Post-request Scripts (Tests)

Write JavaScript to execute after the response is received. Supports:

  • pm.response.json().path.to.field — GJSON data extraction
  • pm.response.headers.get("Header-Name")
  • pm.request.headers.get("Header-Name")
  • Conditional logic with if statements

Note Tab

Add free-form notes or documentation to any request. Notes are persisted and exported with the collection.

4. Response & Validation

JSON Transformation Engine

PostIt features a powerful query engine in the response FILTER bar. You can drill down, search, filter, and transform JSON data on the fly.

Multi-Filtering & Exclusions

Combine multiple filters and exclusions in a single line, separated by spaces:

  • Multiple Exclusions: -**.literals -**.assets -metadata
  • Combined Query: -**.debugInfo data.users[active=true]

Filtering & Searching

  • Drill Down: Standard dot notation. data.user.profile
  • Exclude: Hide specific keys using the minus prefix. -data.logs or -**.password
  • Deep Search: Find a key anywhere. **.id
  • Predicates: Filter arrays by property values. data.users[id=5] or products[price>100]

Projections & Formatting

  • Field Projection: data.users{id, username, email}
  • Flatten View: data | flatten

Transformation Pipes

Chain transformations using the pipe (|) character:

  • | count — Count items in an array or keys in an object
  • | first / | last — First/last item of an array
  • | sort:key — Sort array of objects by key
  • | keys — List all top-level keys

Example Advanced Queries

  • -**.metadata data.items | count
  • **.emails | first
  • data.users[role=admin]{id, name} | sort:name

Response Tabs

  • BODY — Formatted response body with collapsible JSON tree, copy, expand/collapse, and beautify buttons
  • HEADERS — Response headers displayed as key-value pairs
  • PREVIEW — Renders HTML responses in an iframe or displays images
  • JWT — Automatically detects, extracts, and decodes any JSON Web Tokens found in the response body or headers
  • DIFF — Compare the current response side-by-side with a pinned Baseline response
  • SECURITY — Run the Security Fuzzer against the current request

5. History & Time Travel

Persistent Snapshots

Every request sent via the Web UI, TUI, or CLI is automatically recorded in the HISTORY tab with its full context.

  • Full Response Saving: Unlike traditional clients, PostIt saves the exact response body and headers for every historical call.
  • Time Travel: Click any item in the History sidebar to instantly restore that specific response snapshot, even if the server's data has changed since.
  • Metadata: Each record includes the precise timestamp, method, resolved URL, status code, and millisecond duration.

History Actions

  • Visualize Flow — Generate a Mermaid.js sequence diagram from your history items
  • Export HAR — Download history as an HTTP Archive (HAR) file
  • Clear — Clear all history records

Export Formats

  • JSON — /api/history/export?format=json
  • CSV — /api/history/export?format=csv
  • HAR — HTTP Archive format, accessible from the History tab

History Limits

PostIt keeps the last 50 history records. Older records are automatically pruned.

6. Import & Export

Import

Click the Import button (download icon in the sidebar toolbar) to open the import dialog with three tabs:

  • cURL — Paste a cURL command to instantly generate a fully populated request
  • OpenAPI / Swagger — Upload or paste an OpenAPI 3.0+ JSON spec to import all endpoints
  • Postman — Upload or paste a Postman Collection v2.1 JSON to import all requests with headers, bodies, scripts, and folder structure

You can also import collections at startup using the -import flag:

./postit.exe -ui web -import ./my-collection.json,./other-collection.json

Export

  • Postman Collection: Right-click any request or folder and select Export Postman Collection to download a Postman v2.1 compatible JSON file
  • History (JSON/CSV): From the History tab, use the download buttons to export in JSON or CSV format
  • HAR: From the History tab, click Export HAR to download an HTTP Archive file
  • API Docs: Click the External Link icon in the sidebar toolbar, or use /api/docs/generate?format=markdown|html|openapi

7. Environments & The Vault

Environment Switcher

Manage different stages (Local, Staging, Production) using the ENV dropdown in the top toolbar. Click Environments to create and configure environments with named variables.

  • Each environment has a unique ID and a set of key-value variables
  • Variables are resolved in the order: Local (runner) → Active Environment → Global Variables
  • Switch environments instantly to change the variable context for all subsequent requests

The Vault Secure

Mark variables as secret to encrypt them via AES-GCM with a PBKDF2-derived key. Secrets are saved to disk in encrypted form.

  • Unlock: Click the lock icon or use Unlock Vault to enter your master password
  • Encryption: AES-256-GCM with 100,000 PBKDF2 iterations
  • Compatibility: Supports versioned encryption (v1 legacy, v2 PBKDF2)
  • Status: The vault status API at /api/vault/status reports whether the vault is unlocked
  • Encrypted Variables: When locked, secret variables display as [LOCKED]

8. Visual Workflows

Multi-Step API Orchestration

Orchestrate complex multi-step API sequences via the WORKFLOWS sidebar tab. Features an interactive drag-and-drop SVG canvas.

Node Types

  • Request Node: Fire an API call and extract data from the response using GJSON paths into workflow variables
  • Wait Node: Introduce an artificial delay (in milliseconds) between steps
  • Condition Node (If/Else): Branch execution based on GJSON evaluations of the previous response. Routes to success or failure edges.
  • Loop Node: Point to a JSON array path. The engine iterates over each item, injecting it as {{$item}} into downstream request nodes.
  • Script Node: Execute custom JavaScript in the workflow context
  • Input Node: Pause workflow execution and wait for a variable to be set (e.g., wait for an OTP code)

Canvas Features

  • Drag & Drop: Nodes can be repositioned freely on the canvas
  • Edge Drawing: Connect nodes by drawing edges to define execution flow
  • Cycle Detection: The engine detects potential infinite loops (max 1000 visits per node)
  • Execution Logs: View real-time logs as the workflow runs
  • Visualize: Generate Mermaid.js sequence diagrams from workflow execution

Validation

Workflows are validated before execution. Checks include:

  • All referenced nodes exist
  • All request nodes reference valid request paths
  • No excessive edges (potential cycles)
  • Maximum task limit of 5,000 steps to prevent runaway executions

9. Power Tools

9.1 Hammer (Load Testing)

Switch to the Hammer tab in the editor to stress-test the current request. Configure concurrency (Workers) and duration, then click Start Hammering.

  • Live Dashboard: View total requests, RPS (Requests Per Second), and average latency in real-time
  • Percentiles: Automatically calculates P95 and P99 latencies
  • Chart.js Integration: Watch a live graphical plot of your API's performance stability
  • Status Codes: Breakdown of all HTTP status codes received
  • Smart Pre-Resolution: Static variables and headers are resolved once for all workers (configurable per-iteration for dynamic values)

9.2 SQL Sidekick

Run SQL queries immediately after an API call to verify the data layer. Accessible from the SQL tab in the editor.

  • Supported Drivers: SQLite, PostgreSQL, MySQL
  • Auto-Detection: Driver is automatically detected from the connection string format
  • Read-Only Protection: Dangerous keywords (DROP, DELETE, INSERT, UPDATE, etc.) are blocked. Unicode homoglyph bypasses are detected and prevented.
  • Connection Pooling: Database connections are pooled with LRU eviction (max 50 connections, 1 hour max age)
  • Extract to Var: Pull a specific cell from a SQL result (e.g., column id) and save it directly to a variable (e.g., user_id) for the next API call

9.3 WebSocket Client

Connect to and interact with WebSocket servers directly from the WS sidebar tab.

  • URL: Enter ws:// or wss:// URLs
  • Interactive Log: View incoming and outgoing messages with precise timestamps and color-coding
  • Persistent Connection: The connection stays open while you navigate other parts of the app
  • Buffer: Up to 1,000 messages are kept in the buffer
  • Security: Origin validation follows same rules as the main API (localhost, 127.0.0.1, or matching Host header)

9.4 Proxy Mode (The Observer)

PostIt can act as an HTTP Proxy to intercept and record traffic from your browser or mobile applications.

  • Start: Click Start Proxy in the toolbar and choose a port (default 8888)
  • Intercept: Configure your application or OS to use PostIt as its HTTP proxy. All outgoing traffic will be captured and saved as new requests under Intercepted > [Date].
  • Auto-Refresh: The sidebar automatically updates every 5 seconds while the proxy is running
  • SSRF Protection: Private IPs (localhost, 127.0.0.1, AWS metadata, etc.) and non-HTTP/HTTPS schemes are blocked by default
  • Status: The toolbar button toggles between Start Proxy and Stop Proxy

9.5 Intelligent Mock Server

Start PostIt with -mock flag to enable the mock server at /mock/:

./postit.exe -ui web -mock
  • Dynamic Routing: Define paths like /users/:id. PostIt extracts :id and exposes it as a variable for the mock response.
  • Conditional Mocks: Save multiple mocks for one route with GJSON conditions (e.g., id == "5"). The server evaluates the incoming request body and serves the correct response.
  • Simulated Latency: Add a Delay (ms) to any mock to simulate slow network conditions
  • Mock Management: View, add, and delete mocks directly from the Mocks tab in the request editor
  • Stats Dashboard: The MOCK sidebar tab shows hit counts and last access times for all mock endpoints
  • Variable Resolution: Mock response bodies and headers support {{variable}} resolution

9.6 Security Fuzzer

One-click security scan from the SECURITY response tab or the Fuzzer workflow action. Injects payloads into request body fields and query parameters to find vulnerabilities.

  • SQL Injection: Tests with ' OR 1=1 --, '; DROP TABLE --, sleep(5), and more
  • XSS: Tests with <script>alert(1)</script>, javascript:alert(1), SVG/onload vectors
  • Robustness: Edge cases like null, undefined, very large numbers, empty strings
  • Concurrent Scanning: Up to 20 concurrent requests for fast results
  • Results Table: Each payload shows the field targeted, HTTP status code, and response time

9.7 Data-Driven Runner

Execute a request multiple times with different data sets from the Runner tab.

  • Upload Data: Upload a CSV or JSON array file
  • Variable Mapping: File columns/keys are automatically mapped to {{variable_name}} placeholders in the request
  • Local Variables: Each row's data is injected as local variables, taking precedence over environment and global variables
  • Results: An iteration table shows status code, status text, and duration for each run
  • Post-Request Scripts: Test scripts run after each iteration with the local variable context

9.8 GraphQL Explorer

Select the GraphQL body mode to access the dedicated GraphQL editor with schema introspection support.

  • Query Editor: Write GraphQL queries with syntax highlighting
  • Variables Panel: Separate editor for GraphQL variables (JSON format)
  • Schema Introspection: Click Introspect to fetch and browse the remote GraphQL schema (types, fields, arguments, enums, directives)
  • Schema Browser: Navigate the schema tree in the side panel
  • Introspection queries use the standard __schema meta-query with full type references

9.9 Schema Generation & Validation

Generate: Create a JSON Schema from any sample JSON response. The generator infers types (string, number, integer, boolean, array, object) and marks all discovered fields as required.

Validate: Validate any JSON response against a predefined schema. Checks include:

  • Type matching (object, array, primitive)
  • Required field presence
  • Array item type consistency
  • Detailed error messages for each validation failure

API endpoints:

  • POST /api/schema/generate — Generate schema from sample body
  • POST /api/schema/validate — Validate body against schema

9.10 API Documentation Generator

Generate beautiful API documentation directly from your collection. Access via the sidebar toolbar or API.

Formats:

  • Markdown: Clean markdown with headers, code blocks, and request/response details
  • HTML: Self-contained HTML page with dark-themed styling, ready for hosting
  • OpenAPI 3.0: Standard OpenAPI JSON specification for use with Swagger UI, Redoc, etc.

Download via: GET /api/docs/generate?format=markdown|html|openapi

9.11 Code Snippets

Click Code Snippet in the editor toolbar to generate ready-to-use code for any request. Supported languages:

  • JavaScript — Fetch API
  • Go — net/http
  • Python — requests library

9.12 Architecture Visualizer

Generate Mermaid.js sequence diagrams from your request history or workflow executions. Accessible from:

  • History Tab: Click the Visualize Flow button to generate a diagram from selected history records
  • Workflow Logs: Click the Visualize Last Run button after executing a workflow

Sequence diagrams can be exported as SVG for use in documentation, presentations, or wikis.

10. CLI Runner (CI/CD)

Execute your collections entirely headlessly from the terminal for automation pipelines:

./postit.exe run --collection ./output/history.json --env ./output/environments.json --delay 500

Flags:

  • --collection — Path to the collection JSON file (required)
  • --env — Path to an environment JSON file (optional)
  • --delay — Delay between requests in milliseconds (optional)

The CLI runner:

  • Loads the environment if provided (sets it as the active environment)
  • Executes requests in the exact order defined in the collection
  • Runs pre-request and post-request scripts for each request
  • Prints colored output (green for 2xx/3xx, red for 4xx/5xx)
  • Exits when all requests have been processed

11. Terminal UI (TUI)

PostIt includes a full Terminal User Interface for use in SSH sessions or terminal-only environments:

./postit.exe -ui tui

The TUI provides:

  • Request list navigation with keyboard shortcuts
  • Inline request editing
  • Response viewing
  • Variable and global header management
  • Create, edit, duplicate, and delete requests
